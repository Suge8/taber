# 产品

## 注册登记

product

## 用户

使用 Chrome 或 Edge、希望有一个强大全能的侧边栏浏览器 Agent 来阅读页面、收集信息、填写表单、处理和生成文档（PDF/Word）、解释页面故障的人。

## 产品目的

Taber 从一个 offscreen AgentHost 运行浏览器 Agent，使用小而固定的工具边界，将事件持久化到 Dexie 中，并从该日志重建侧边栏，使得工作在 UI 关闭后仍能继续。

## 品牌人格

冷静、精确、负责任。界面应该让人感觉像一个值得信赖的操作员控制台，而不是一个聊天机器人玩具或开发者调试界面。

## 反面参照

不要看起来像一个隐身自动化工具、反检测产品、逐步审批系统、原始 DevTools 克隆，或装饰性 AI SaaS 仪表盘。不要暴露原始思维链。

## 设计原则

- 在装饰之前先展示工具证据与任务状态。
- 通过可见的停止与恢复状态让用户保持掌控。
- 把 Dexie 事件历史当作 UI 的唯一事实源。
- 工具边界保持狭窄：文档、图片、导航、页面操作、REPL、文件系统；debugger 仅在 debug 构建中可用。
- 偏好密集的清晰度，而非营销式的炫技。

## 无障碍与包容性

默认跟随系统主题，支持浅色与深色主题，保留可读对比度，避免在狭窄的侧边栏中溢出，并尊重减弱动效偏好。
